LLM-Powered Search System

A semantic search system for Kita Hub using RAG pipeline

Project Overview

Developed a fully functional AI-powered semantic search prototype for Kita Hub, using a Retrieval-Augmented Generation (RAG) pipeline to deliver accurate, source-linked answers from internal documents in German. The system supports both platform-related and training-related queries for Kita workers.

The prototype was built using ~80 curated German PDF documents and combines semantic search, fine-tuned LLMs, and modern frontend development aligned with corporate design.

Key Features

  • RAG Pipeline: Integrated document retrieval with LLM-based answer generation for precise and transparent results.
  • Vector Search with FAISS: Used Hugging Face embeddings and FAISS for efficient semantic search over PDF content.
  • German-Language LLM: Deployed a LLaMA-based model fine-tuned for German to ensure high relevance and contextual accuracy.
  • Source Attribution: Each response includes direct links to the original source document.
  • Supports Two Query Types:
    • Platform usage (e.g., mail notifications, account settings)
    • Educational content for Kita workers

Technologies Used

Backend

  • Python, FastAPI
  • Sentence Transformers (all-MiniLM-L6-v2)
  • FAISS for vector-based semantic search
  • PDFPlumber & PyTesseract for text and OCR extraction
  • Private LLM API (German-tuned, LLaMA-based) for response generation
  • Langdetect for detecting query language
  • Docker for containerization

Frontend

  • HTML, CSS, JavaScript
  • Fetch API for async communication

Architecture & Data

Kita Speech Translator

A comprehensive web application for real-time speech translation and processing

Project Overview

Kita Speech Translator is a comprehensive web application that provides real-time speech-to-text, translation, and text-to-speech capabilities across multiple languages. The system is designed with three distinct processing pipelines optimized for different use cases:

  • Pipeline 1 (Direct Processing): Handles microphone recordings with immediate processing for quick conversational translation.
  • Pipeline 2 (Small File Processing): Processes audio files under 6 minutes with simple, direct translation without progress tracking.
  • Pipeline 3 (Large File Processing): Handles longer audio files (>6 minutes) by breaking them into 1-minute chunks and processing them in parallel with 6 workers, showing incremental results and detailed progress tracking.

The application features a user-friendly interface that allows users to record audio directly from their browser or upload audio files in various formats. The system automatically determines the appropriate processing pipeline based on the input type and file size, providing optimal performance for each scenario.

Key Features

  • Multi-language Support: Translates between 60+ languages with automatic language detection capability.
  • Adaptive Processing Pipeline: Automatically selects the most efficient processing approach based on input type and duration.
  • Real-time Incremental Translation: Shows translation results as they become available, particularly useful for longer files.
  • Parallel Processing: Utilizes parallel workers to efficiently process larger files by breaking them into manageable chunks.
  • Progress Tracking: Provides detailed progress indicators for large file processing with estimated completion times.
  • Text-to-Speech (TTS): Converts translated text to natural-sounding speech in the target language.
  • Conversation History: Maintains and displays the full conversation history with options to download transcripts.

Technologies Used

Backend

  • Python with FastAPI for the RESTful API framework
  • Whisper models for speech-to-text transcription
  • Marian-MT for neural machine translation
  • Coqui TTS for text-to-speech synthesis
  • Asyncio for concurrent processing
  • Docker for containerization and deployment

Frontend

  • Modern JavaScript (ES6+) for client-side logic
  • HTML5/CSS3 for responsive layout
  • Web Audio API for recording functionality
  • Fetch API for asynchronous communication with the backend

Processing Technologies

  • FFMPEG for audio preprocessing and conversion
  • PyDub and SoundFile for audio manipulation
  • Hugging Face Transformers for ML model integration
  • Multithreaded processing for performance optimization

Impact

Kita Speech Translator bridges language barriers in real-time, enabling effective communication across language boundaries. The system's adaptive processing approach allows it to be used in various scenarios, from quick one-on-one conversations to processing longer presentations or meetings.

The ability to operate in air-gapped environments makes it suitable for use in secure facilities where external network access is restricted. The containerized deployment simplifies installation and ensures consistent operation across different environments.

Overall, Kita Speech Translator represents a powerful tool for multilingual communication, language learning, content localization, and international collaboration, with its tiered processing approach offering an optimal balance between performance and resource utilization.

Analyzing Earthquake Patterns in 2023

Comprehensive analysis and visualization of seismic activity in 2023

Project Overview

Earthquakes 2023 is a data analysis and visualization project focused on seismic activity patterns throughout the year 2023. Using real-world earthquake data sourced from USGS and Kaggle, the project explores spatial and temporal distributions, magnitudes, depths, and underlying correlations between key seismic parameters.

Through a combination of statistical analysis, geospatial mapping, clustering, and time-series techniques, the project uncovers actionable insights that support disaster preparedness and seismic risk evaluation.

Key Features

  • Exploratory Data Analysis (EDA): Used scatterplots, heatmaps, pair plots, and categorical plots to reveal trends and relationships in seismic data.
  • Geospatial Mapping: Leveraged shapefiles and geographic visualizations to map earthquake locations and intensities in relation to tectonic plates.
  • Regression Analysis: Investigated relationships between magnitude, depth, and reporting stations.
  • Clustering: Applied unsupervised learning to detect regional seismic activity patterns.
  • Time-Series Analysis: Analyzed temporal trends to identify peak activity periods and shifts over time.
  • Interactive Dashboards: Presented key insights through Tableau dashboards to support non-technical audiences.

Technologies Used

  • Languages: Python
  • Libraries: Pandas, NumPy, Seaborn, Matplotlib, Plotly
  • Machine Learning: Scikit-learn (clustering, regression)
  • Geospatial Tools: GeoPandas, Shapely
  • Visualization: Tableau (for interactive dashboards)
  • Data Source: USGS Earthquake Catalog (via Kaggle)

Impact

  • Identified correlations between earthquake magnitude and seismic station reporting density.
  • Mapped seismic activity relative to tectonic boundaries for improved spatial awareness.
  • Discovered patterns in magnitude and depth distributions relevant for hazard forecasting.
  • Enabled interactive exploration of earthquake data via Tableau dashboards.
